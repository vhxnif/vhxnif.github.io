生命周期

Maven主要有以下 3 种内置的生命周期 liftcycle。

clean

用于清理maven产生的文件、文件夹、包含一个 clean 阶段。

site

用于处理项目文档的创建，包含一个 site 阶段，该阶段会生成项目的文档。

default

用于处理项目的构建和部署，包括以下阶段：

  1. validate 验证项目正确无误，且所有必要信息均已齐备。

  2. compile 编译项目的源代码。

  3. test 使用适当的单元测试框架对编译后的源代码进行测试。这些测试不应要求代码被打包或部署。

  4. package 将编译后的代码打包成可分发的格式，例如，jar文件。

  5. verify 对集成测试结果进行任何检查，以确保满足质量标准。

  6. install 将包安装到本地仓库，以便在本地其他项目中用作依赖项。

  7. deploy 将最终的包复制到远程仓库。

以上只是部分生命周期，其他的生命周期查看文档。

不同的构建目标有不同的 default 默认生命周期。构建 jar 包的标准生命周期阶段( phase )如下：

Plugins

Apache Maven Resource Plugin

负责将项目资源复制到输出目录，项目资源分为两种：main resources 和 test resources。

该插件包含三个 goal:

  1. resources:resources 将mian包下的源代码复制到指定的输出目录。通常会自动执行，其默认绑定在 process-resources 阶段，始终使用 project.build.resources 元素来指定资源，并且默认使用 project.build.outputDirectory 来指定输出目录。

  2. resources:testResources 将test包下面的代码复制到指定的输出目录。通常会自动执行，默认绑定在 process-test-resources 阶段，始终使用 project.build.testResources 元素来指定资源，并且默认使用 project.build.testOutputDirectory 来指定输出目录。

  3. resources:copy-resources 将资源复制到输出目录。需要自己指定需要复制的资源目录以及输出目录。

Apache Maven Surefire Plugin

Surefire 插件在构建的 test 阶段用于执行应用程序的单元测试。

生成两种不同格式的测试报告：

  1. 纯文本文件 *.txt

  2. XML文件 *.xml

默认情况下，测试报告会生成在 ${basedir}/target/surefire-reports/TEST-*.xml。

Surefire 只有一个 goal:

surefire:test 运行项目的单元测试。
